# Round 5 主线程跟踪清单

> 主 agent 使用

## 1. 当前轮次冻结项

- 冻结目标：多租户境外 DID 语音平台
- 冻结旧模式定位：`direct_prefix_mode` 仅为兼容模式
- 冻结客户端方向：新模式必须走服务端编排抽象
- 冻结数据隔离：共享库 + 行级租户隔离
- 冻结回拨实现：`display_did + remote_number + TTL`

## 3. 集成检查顺序

1. 先看是否遵守写入所有权
2. 再看是否违反真源
3. 再看是否埋下跨线程返工点
4. 最后才看实现细节完整度

## 4. 主线程重点盯防项

- 子线程是否把旧的 `#31#` 模式继续当主路径扩写
- 子线程是否跳过租户建模直接写平台级表
- 子线程是否在没有契约冻结前硬编码 trunk/webhook 字段
- 子线程是否跨目录抢写

## 5. 下一轮集成门槛

只有在以下条件都满足时，主线程才推进第二轮：

- A 线程：核心多租户与语音实体已成型
- B 线程：客户端与服务端的出呼抽象已存在
- C 线程：事件和 webhook 契约已冻结
- D 线程：联调门禁和回归模板已到位

# Round 1 主线程跟踪清单

## 1. 当前运行中的子线程

## 2. 主线程验收顺序

主 agent 收到结果后，按以下顺序验收：

1. 是否遵守目录写入边界
2. 是否覆盖任务卡要求的输出物
3. 是否显式列出假设、未完成项与阻塞点
4. 是否为第二轮联调提供可直接复用的底座
5. 是否引入了越权决策或超出首发范围的扩张

## 3. 验收清单

### 3.1 Client Worker

- 仅修改 `client/`
- 有 Flutter 工程骨架
- 有路由、登录态守卫、页面壳
- 有本地存储与拨号域抽象
- 明确 mock 点与后端接入点

### 3.2 Server Worker

- 仅修改 `server/`
- 有 NestJS 工程骨架
- 有核心表结构、迁移或等效数据库底座
- 有 `auth/config` 模块最小骨架
- 明确接口契约、假设与阻塞点

### 3.3 QA/Release Worker

- 仅修改 `docs/qa/`
- 有测试策略
- 有接口联调用例
- 有冒烟/回归清单
- 有兼容矩阵模板
- 有发布检查表与风险清单

## 4. 第二轮拆分前置条件

只有当以下条件满足时，主 agent 才进入第二轮任务拆分：

- 客户端与服务端都已形成各自最小骨架
- 目录边界没有被破坏
- 接口与数据模型没有出现冲突性定义
- QA 已经形成阶段 1 的联调与发布准入基线

## 5. 第二轮候选主题

如果第一轮通过，第二轮优先拆分：

1. 登录联调
2. 配置联调
3. 拨号闭环与异常回退
4. 最小测试验证

# 需求追踪矩阵

## 1. 说明

本矩阵用于把阶段 1 的测试点和发布检查项，逐条追溯到现有方案、设计和开发计划文档，避免测试资产脱离需求来源。

## 2. 追踪规则

| 字段 | 说明 |
|------|------|
| 需求 ID | QA 统一编号 |
| 来源 | 方案书 / 设计文档 / 开发计划文档 / AgentTeams |
| 需求摘要 | 可验证、可执行的需求描述 |
| 覆盖物 | 对应的接口用例、冒烟、回归、兼容、发布检查 |
| 优先级 | P0 / P1 / P2 |
| 责任归属 | Client / Server / QA / 主 agent / 产品 |

## 4. 需求到测试的最小映射

| 需求域 | 必测项 | 最低产物 |
|--------|--------|----------|
| 登录 | 发送验证码、验证码登录、刷新令牌、退出登录、未登录拦截 | 接口用例 + 冒烟清单 |
| 配置 | 前缀列表、公告列表、默认前缀持久化 | 接口用例 + 回归清单 |
| 拨号 | 普通拨打、隐私拨打、号码校验、前缀拼接、失败回退 | 冒烟清单 + 回归清单 |
| 兼容 | Android/iOS、运营商、系统版本、前缀组合 | 兼容矩阵模板 |
| 发布 | 预生产、监控、回滚、签核 | 发布检查表 + 风险预案 |

## 5. 需要额外确认的项

| 项目 | 原因 | 责任归属 |
|------|------|----------|
| 测试验证码获取方式 | 设计文档未定义测试白名单或 Mock 方式 | Server Worker + 主 agent |
| 预生产域名与环境变量 | 影响联调与发布检查 | Server Worker + 运维 |
| 真机与 SIM 卡清单 | 影响兼容矩阵完整度 | QA / Release |
| 具体前缀最终名单 | 影响配置和兼容验证范围 | 主 agent + 产品 |

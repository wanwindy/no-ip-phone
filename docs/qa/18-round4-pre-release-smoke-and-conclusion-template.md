# Round 4 预发布 Smoke 清单与结论模板

## 1. 适用时机

本清单用于候选包冻结后、进入发布建议前的最后一轮预发布 smoke。它不是功能回归全集，而是回答“这个版本现在能不能发”的最小检查。

使用要求：

1. 先完成 [16-round4-minimum-real-device-coverage-plan.md](./16-round4-minimum-real-device-coverage-plan.md) 的必测项，再执行本清单。
2. smoke 结论只基于真实执行结果，不允许预填“通过”。
3. 如果执行过程中换包、换环境或换配置，必须重新开一份 smoke 记录。

### 7.2 结论正文

按下面格式填写，避免漏写重点：

1. 本轮执行范围：`Android/iOS 样本、运营商覆盖、是否包含回退与文案复核`。
2. 通过情况：`阻塞级通过 x 项，非阻塞级通过 x 项`。
3. 问题情况：`新增问题 x 个，其中 P0/P1/P2/P3 各多少`。
4. 风险说明：`哪些机型、运营商、入口受限，是否已有替代路径`。
5. 放行建议：`建议发布 / 有条件发布 / 暂不建议发布 / 明确阻塞发布`。
6. 若不是“建议发布”：`必须写清阻塞项、责任人、下次复测时间`。

## 8. 禁止写法

以下写法在 Round 4 smoke 结论中禁止出现：

1. “大概率没问题，可直接发。”
2. “未测 iOS，但参考 Android 结果判断应该通过。”
3. “运营商资源不够，先按通过处理。”
4. “帮助页还没改，但先不影响发布。”

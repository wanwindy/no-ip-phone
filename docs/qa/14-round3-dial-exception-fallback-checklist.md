# Round 3 拨号异常与回退验证清单

## 1. 目的

本清单用于验证拨号体验在异常条件下的表现，重点关注号码非法、设备不支持、拨号器拉起失败、用户取消、帮助提示与能力边界文案是否一致。

## 2. 验证原则

1. 只验证真实回退，不伪造结果。
2. 每个异常项都要记录输入、触发点、提示内容和回退结果。
3. 如果系统行为和文案不一致，优先记录差异，不要自行解释。
4. 如果需要引用 Round 2 结论，仍然保持 `logout` 语义不变。

## 4. 结果判定

| 判定 | 含义 | 处理方式 |
|------|------|----------|
| 通过 | 异常分支行为符合预期 | 回填到兼容矩阵与日报 |
| 部分通过 | 有提示但回退不完整，或文案与行为有细微差异 | 标为需优化，附证据 |
| 不通过 | 异常处理失败、无提示或误导用户 | 形成缺陷 |
| 待确认 | 现场无法判定 | 补证据后复核 |

## 5. 记录要求

1. 每个异常项都必须有一条独立记录。
2. 如果问题与机型有关，必须写清机型和系统版本。
3. 如果问题与运营商或网络制式有关，必须写清 SIM 和网络环境。
4. 如果同一异常重复出现三次以上，建议单独升级为阻塞项候选。

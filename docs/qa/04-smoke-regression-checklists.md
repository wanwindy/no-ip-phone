# 冒烟清单与回归清单

## 1. 冒烟清单

冒烟清单用于每次接口联调、候选包验证和预生产验收前的快速判断。

| ID | 场景 | 步骤 | 预期结果 | 优先级 |
|----|------|------|----------|--------|
| SMK-001 | 未登录守卫 | 清空登录态并启动 App | 自动进入登录页，不能直达主页或拨号页 | P0 |
| SMK-002 | 登录成功 | 输入手机号，完成验证码登录 | 进入主页，登录态持久化 | P0 |
| SMK-003 | 普通拨打 | 输入合法号码，点普通拨打 | 调起系统电话 App，拨号串不带隐私前缀 | P0 |
| SMK-004 | 隐私拨打 | 输入合法号码，点隐私拨打 | 调起系统电话 App，拨号串包含当前前缀，默认 `#31#` | P0 |
| SMK-005 | 风险提示 | 首次使用隐私拨打 | 展示运营商影响提示，可继续或取消 | P1 |
| SMK-006 | 前缀设置 | 修改默认前缀并保存 | 重启后仍保留设置 | P0 |
| SMK-007 | 帮助页 | 打开帮助页 | 显示能力边界与常见说明 | P1 |
| SMK-008 | 退出登录 | 点击退出登录 | 清除登录态并回到登录页 | P0 |

## 2. 回归清单

回归清单用于每次修复后或版本封板前执行，覆盖高风险路径和容易回归的地方。

| ID | 回归点 | 验证内容 | 预期结果 | 优先级 |
|----|--------|----------|----------|--------|
| REG-001 | 路由守卫 | 退出登录后访问受保护页 | 被重定向到登录页 | P0 |
| REG-002 | Token 刷新 | 访问受保护接口时 token 过期 | 客户端自动刷新并继续请求 | P0 |
| REG-003 | 刷新轮换 | 刷新后旧 refresh token 失效 | 旧 token 不可继续使用 | P0 |
| REG-004 | 号码清洗 | 输入带空格、横杠、括号号码 | 清洗后仍能正确校验和拨号 | P0 |
| REG-005 | 前缀优先级 | 用户手动前缀覆盖默认前缀 | 最终拨号串使用用户选择 | P0 |
| REG-006 | 文案边界 | 风险说明、帮助说明、公告文案 | 不出现“100% 隐藏”等误导表述 | P0 |
| REG-007 | 本地存储 | 重启后 token、前缀、已读提示 | 状态按设计保持 | P1 |
| REG-008 | 失败回退 | 拨号器不可用或 launch 失败 | 给出错误提示并保留普通拨打路径 | P0 |
| REG-009 | 配置加载 | 前缀/公告接口失败 | 采用可解释的降级或提示 | P0 |
| REG-010 | 登录频控 | 多次发送验证码/多次输错 | 正确触发限流或锁定提示 | P0 |

## 3. 执行顺序

1. 先执行 `SMK-001` 到 `SMK-008`。
2. 冒烟通过后再做 `REG-001` 到 `REG-010`。
3. 任一 `P0` 回归失败都不建议进入预生产验收。

## 4. 判定标准

| 结果 | 判定 |
|------|------|
| 通过 | 预期与实际一致，无阻断缺陷 |
| 失败 | 预期与实际不一致，或出现阻断问题 |
| 需要跟进 | 有可接受降级，但必须记录责任人和修复计划 |

# 预生产验收与发布检查表

## 1. 预生产验收目标

确认候选版本已经具备上线条件，且在预生产环境中完成主流程、兼容性、监控和回滚准备验证。

## 2. 发布前检查

| 检查项 | 通过标准 | 责任归属 |
|------|----------|----------|
| 版本冻结 | 候选版本号、构建号、变更范围已确认 | 主 agent + 项目 |
| 接口稳定 | `auth/config` 契约无未同步变更 | Server Worker |
| 客户端稳定 | 登录、拨号、设置、帮助已完成回归 | Client Worker |
| 兼容矩阵 | 关键机型/系统/运营商组合已填报 | QA / Release |
| 文案边界 | 不存在“100%隐藏”或“必然成功”表述 | 主 agent / 产品 |
| 环境变量 | 预生产配置、密钥、域名已准备 | Server Worker + 运维 |
| 监控告警 | 错误监控、日志、基础告警可用 | 运维 + QA |
| 回滚方案 | 回滚包、回滚条件、负责人明确 | QA / Release + 运维 |

## 3. 预生产验收步骤

| 步骤 | 内容 | 预期 |
|------|------|------|
| 1 | 部署候选版本到预生产环境 | 服务可启动，接口健康 |
| 2 | 执行 `API-001` 到 `API-024` 关键用例 | 主链路全部通过 |
| 3 | 执行 `SMK-001` 到 `SMK-008` | 冒烟通过 |
| 4 | 选取关键设备跑兼容矩阵 | 形成可用/限制/不通过结论 |
| 5 | 校验日志与监控 | 出现异常可定位、可告警 |
| 6 | 演练回滚 | 能在约定时间内恢复到上一版本 |

## 4. 发布门槛

| 门槛 | 条件 |
|------|------|
| 功能门槛 | 登录、拨号、配置、帮助、退出可用 |
| 质量门槛 | P0 缺陷清零，P1 有明确降级或延期方案 |
| 风险门槛 | 兼容限制已明确，文案无误导 |
| 运维门槛 | 监控、日志、回滚路径可用 |
| 责任门槛 | 发布负责人、值守负责人、升级路径已明确 |

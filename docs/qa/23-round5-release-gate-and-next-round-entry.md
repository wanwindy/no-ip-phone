# Round 5 发布门禁与下一轮准入清单

> 日期：2026-04-22  
> 适用轮次：Round 5 / 第一轮收口、Round 5 / 第二轮启动前  
> 目标：把第一轮输出转成可勾选的发布门禁和下一轮启动条件，避免主线程在不完整状态下推进集成。

## 1. 结论等级

| 结论 | 含义 | 使用条件 |
|------|------|----------|
| GO | 允许进入主线程集成或下一轮 | 所有 P0 门禁通过，P1 有明确 owner 且不影响高风险区 |
| CONDITIONAL GO | 可以进入受控集成，不允许对外宣称稳定 | 无 P0，但仍有可接受 P1/P2 且已有时限与责任人 |
| NO-GO | 不允许进入下一轮 | 出现 P0、未冻结契约变化、证据缺失或高风险区未覆盖 |

## 2. Round 5 第一轮发布门禁

| ID | 门禁项 | 对应任务卡 | 通过标准 | 失败口径 |
|----|--------|------------|----------|----------|
| GATE-01 | 真源一致性 | A, B, C, D | 输出物不违背四份真源 | 任一线程把旧模式重新当主路径 |
| GATE-02 | 多租户核心模型成型 | A | `tenants`、`tenant_members`、`did_inventory`、`tenant_did_assignments`、`call_sessions`、`callback_sessions` 已有最小可用骨架 | 核心表缺失或租户边界不清 |
| GATE-03 | 租户隔离验证通过 | A, B, D | 跨租户读写、串号、串回拨均为零 | 任一跨租户泄漏 |
| GATE-04 | 出呼任务 API 壳可联调 | B, C, D | `POST /calls/outbound` 可按冻结契约返回占位状态 | 字段不齐、字段名漂移、无差异记录 |
| GATE-05 | 双模式拨号抽象成立 | B, D | `direct_prefix_mode` 与 `server_orchestrated_mode` 都可区分执行 | 两条链路混淆或新模式被旧模式替代 |
| GATE-06 | DID 外显字段一致 | A, B, C, D | 数据层、API 层、状态占位中展示 DID 一致 | 展示 DID 不一致或无来源 |
| GATE-07 | 同号回拨规则通过 | A, C, D | 正向命中正确，负向误命中为零 | 错 DID、错号码、过期 TTL 仍命中 |
| GATE-08 | 契约冻结完成 | C, D | event / webhook / call-state / callback 规则均有冻结文档 | 有待定字段或消费者口径不一 |
| GATE-09 | QA 基线完整 | D | 追踪矩阵、联调清单、冒烟回归模板、发布门禁齐备 | 任何一份缺失或不可执行 |
| GATE-10 | 缺陷与阻塞可追踪 | D | 所有遗留问题有 owner、优先级、下一步 | 存在无 owner 的高风险缺陷 |

## 3. 高风险区放行口径

| 高风险区 | 必过项 | 允许遗留 | 不允许遗留 |
|----------|--------|----------|------------|
| 租户隔离 | `SMK-01`、`SMK-02`、`INT-02` | 非阻断文档优化项 | 任何跨租户数据泄漏 |
| 回拨命中 | `SMK-05`、`SMK-06`、`INT-08`、`INT-09` | 非默认 TTL 文案待细化 | 错误命中、TTL 失效 |
| DID 外显 | `SMK-04`、`INT-03`、`INT-05` | 真实公网验证待下一轮 | API / UI / 数据展示号不一致 |
| 双模式切换 | `SMK-07`、`SMK-08`、`INT-06`、`INT-10` | 兼容模式说明文案优化 | 新模式被绕开、旧模式成为默认 |
| API 契约变更 | `SMK-03`、`SMK-09`、`INT-04`、`INT-11` | 非结构化文档润色 | 无记录的破坏性字段变更 |

## 4. 下一轮启动门禁

| ID | 启动条件 | Owner | 通过标准 |
|----|----------|-------|----------|
| NEXT-01 | A/B/C/D 第一轮输出已进入统一索引 | 主线程, D | 主线程能按索引直接拉起第二轮 |
| NEXT-02 | 四张任务卡的最小完成定义均已满足 | 主线程 | `docs/Round5主线程跟踪清单.md` 第 5 节可逐条对上 |
| NEXT-03 | 双租户、双 DID、双模式测试数据可复用 | A, B, D | 下一轮不需要重新造数据口径 |
| NEXT-04 | `/calls/outbound` 字段与 callback / event 字段已冻结 | B, C | 第二轮实现不再猜字段 |
| NEXT-05 | 回拨 TTL、命中键、失败口径已冻结 | C, D | 第二轮联调直接复用本轮口径 |
| NEXT-06 | 高风险区无未分配 owner 的遗留问题 | 主线程, D | 每个问题都有责任人和计划日期 |
| NEXT-07 | 兼容模式保留策略已明确 | B, 主线程 | `direct_prefix_mode` 保留但不扩大职责 |
| NEXT-08 | 下一轮集成顺序已确认 | 主线程 | 先 A/B/C 集成，再执行 D 的门禁验证 |

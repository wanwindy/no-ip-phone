# 阶段 1 测试策略 v1

## 1. 目标

阶段 1 的目标是验证首发主链路是否可用：`登录 -> 取配置 -> 进入主页 -> 普通拨打/隐私拨打 -> 设置与帮助可用 -> 退出与回退可用`。

## 2. 测试范围

| 范围 | 覆盖内容 | 依据 |
|------|----------|------|
| 登录鉴权 | 手机号验证码登录、刷新令牌、退出登录、用户信息 | [方案书.md](../方案书.md) §10、[设计文档.md](../设计文档.md) §6 |
| 拨号主链路 | 号码清洗、普通拨打、隐私拨打、前缀拼接、系统电话 App 调起 | [方案书.md](../方案书.md) §11、[设计文档.md](../设计文档.md) §3.4、§7 |
| 配置能力 | 前缀列表、公告列表、默认前缀读取 | [方案书.md](../方案书.md) §10.5、[设计文档.md](../设计文档.md) §6.3 |
| 页面与守卫 | 启动页、登录页、主页、设置页、帮助页、未登录拦截 | [方案书.md](../方案书.md) §12、[开发计划文档.md](../开发计划文档.md) §5.1 |
| 可用性与边界 | 登录后才能拨号、效果受运营商/机型影响、普通拨打回退 | [方案书.md](../方案书.md) §3、§9、§19 |

## 3. 测试目标

1. 证明未登录用户不能进入拨号主流程。
2. 证明登录流程、令牌刷新和退出登录闭环可用。
3. 证明普通拨打和隐私拨打都能正确拼接并调起系统电话能力。
4. 证明前缀、帮助、提示和回退路径可用。
5. 证明发布前已识别主要兼容风险，并完成可追踪验证。

## 4. 测试层级

| 层级 | 重点 | 产出 |
|------|------|------|
| 接口联调 | auth/config 主接口契约验证 | 接口用例结果、缺陷单 |
| 冒烟测试 | 登录、主页、拨号、设置、帮助 | 冒烟通过记录 |
| 回归测试 | 鉴权、路由、前缀、异常提示、历史与退出 | 回归清单结果 |
| 兼容测试 | 机型、系统、运营商、网络模式、前缀组合 | 兼容矩阵 |
| 预生产验收 | 环境、配置、监控、回滚、发布门槛 | 发布签核建议 |

## 5. 准入条件

| 条件 | 判定标准 | 责任归属 |
|------|----------|----------|
| 接口契约冻结 | `/auth/*` 与 `/config/*` 字段、错误码、鉴权方式不再变更 | 主 agent + Server Worker |
| 客户端基础流可跑 | 启动页、登录页、主页壳、设置页、帮助页可进入 | Client Worker |
| 测试环境可用 | QA 环境可访问，基础联调用例可执行 | QA / Release + 运维 |
| 真机资源到位 | 至少准备 Android、iOS 真机和可用 SIM 卡 | QA / Release |
| 产品边界已确认 | 文案明确“尝试隐藏”，不承诺 100% | 主 agent / 产品 |

## 6. 准出标准

| 标准 | 判定标准 |
|------|----------|
| P0 用例 | 全部通过，无阻塞缺陷 |
| P1 用例 | 允许少量低风险问题，但不影响登录、拨号和发布判断 |
| 兼容矩阵 | 至少覆盖优先机型、系统、运营商和默认前缀组合 |
| 发布材料 | 预生产检查表、风险清单、回滚预案齐备 |
| 缺陷状态 | 关键阻塞已关闭或已形成明确降级方案 |

## 7. 缺陷分级

| 级别 | 定义 | 处理要求 |
|------|------|----------|
| P0 | 阻断登录、鉴权、拨号、发布或造成数据/合规风险 | 立即修复，禁止带缺陷发布 |
| P1 | 影响主流程但有稳定回退方案 | 修复后复测，必要时降级发布范围 |
| P2 | 影响体验但不阻断主流程 | 进入迭代修复 |
| P3 | 文案、样式、轻微兼容问题 | 记录即可 |

## 8. 质量策略

### 8.1 接口优先

先验证 `auth/config` 契约，再做页面联调。这样可以尽早发现字段、鉴权和错误码问题，避免后期客户端和服务端各自实现后再返工。

### 8.2 主流程优先

测试顺序固定为：登录 -> 取配置 -> 主页 -> 普通拨打 -> 隐私拨打 -> 设置 -> 退出登录 -> 回到登录守卫。任何主流程失败都应优先修复。

### 8.3 风险前置

隐私拨打的结果受运营商和终端影响，不能用“是否真实隐藏成功”作为单一上线条件。阶段 1 只要求客户端正确拼接前缀、正确提示边界、正确提供回退路径。

### 8.4 可追踪

所有测试点必须在 [02-requirements-traceability.md](./02-requirements-traceability.md) 中找到对应需求来源、测试类型和责任归属。

## 9. 第一轮重点风险

| 风险 | 影响 | 处理 |
|------|------|------|
| 验证码/登录链路不稳定 | 阻断主入口 | 优先做接口联调与失败码验证 |
| 拨号器对 `tel:` 或 `#31#` 行为不一致 | 影响隐私拨打结果 | 以矩阵验证为准，保留普通拨打 |
| 未登录守卫失效 | 违反产品边界 | 冒烟和回归必须覆盖 |
| 前缀配置无法读取 | 隐私拨打配置失效 | 先验证接口，再验证本地缓存 |
| 真机/运营商资源不足 | 兼容结论不充分 | 预生产前锁定测试资源 |

## 10. 输出物

1. 接口联调用例执行记录。
2. 冒烟与回归结果清单。
3. 兼容矩阵模板和填报结果。
4. 预生产验收建议。
5. 风险、阻塞和回滚预案。

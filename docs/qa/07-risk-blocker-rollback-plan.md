# 风险、阻塞与回滚预案清单

## 1. 风险台账

| 风险 ID | 风险 | 影响 | 等级 | 责任归属 | 缓解措施 | 回滚/降级 |
|---------|------|------|------|----------|----------|----------|
| RSK-001 | 运营商不支持 `#31#` | 隐私拨打不生效 | 高 | 产品 + QA + 主 agent | 帮助页与提示文案明确边界，保留普通拨打 | 禁止把“隐藏成功率”作为上线硬门槛 |
| RSK-002 | 系统拨号器对 `tel:` 或 `#` 处理异常 | 拨号失败 | 高 | Client Worker | 真机矩阵提前验证，保留回退提示 | 使用普通拨打或限制受影响机型 |
| RSK-003 | 验证码服务不稳定 | 无法登录 | 高 | Server Worker | 发送限流、供应商稳定性确认、Mock 兜底 | 暂停发布，恢复登录链路后再上线 |
| RSK-004 | 前缀配置接口不可用 | 隐私拨号配置失效 | 中 | Server Worker | 本地默认前缀兜底 | 使用默认 `#31#` 并提示配置暂不可用 |
| RSK-005 | 兼容矩阵覆盖不足 | 发布判断不充分 | 高 | QA / Release | 预先锁定设备和 SIM | 延后发布，不做盲发 |
| RSK-006 | 文案误导用户 | 合规与投诉风险 | 高 | 主 agent / 产品 | 文案冻结，评审口径一致 | 立即修正文案后再发布 |

## 2. 阻塞项清单

| 阻塞 ID | 阻塞项 | 说明 | 责任归属 | 处理建议 |
|---------|--------|------|----------|----------|
| BLK-001 | 测试验证码方案未定 | 没有白名单或 Mock，联调无法闭环 | Server Worker + 主 agent | 先确认测试账号/验证码机制 |
| BLK-002 | 预生产环境未就绪 | 无法验收和回滚演练 | 运维 + Server Worker | 先完成环境、域名、密钥与数据库准备 |
| BLK-003 | 真机/SIM 资源不足 | 无法形成兼容结论 | QA / Release | 先锁定最小设备池，再扩展 |
| BLK-004 | 接口契约变更未同步 | 客户端和测试用例失配 | 主 agent + Server Worker | 冻结接口后再联调 |
| BLK-005 | 产品能力边界未冻结 | 可能出现误导发布 | 主 agent / 产品 | 先统一话术再进入发布准备 |

## 3. 回滚预案

### 3.1 触发条件

| 条件 | 触发动作 |
|------|----------|
| 登录失败率异常升高 | 暂停扩大发布范围 |
| 拨号器调起失败率异常升高 | 立即转入回滚判断 |
| 预生产冒烟失败 | 阻止发布 |
| 兼容矩阵出现高风险不通过 | 仅允许限制范围发布或延期 |
| 监控无法观测关键指标 | 先补齐监控再发布 |

### 3.2 回滚动作

| 场景 | 动作 | 责任归属 |
|------|------|----------|
| 客户端版本问题 | 回退到上一稳定包 | Client Worker + QA |
| 服务端接口问题 | 回退服务端镜像或停止放量 | Server Worker + 运维 |
| 文案/配置问题 | 回退配置或公告内容 | 主 agent + Server Worker |
| 运营商兼容问题 | 限制支持范围，更新帮助页说明 | QA / Release + 主 agent |

### 3.3 回滚负责人

| 角色 | 责任 |
|------|------|
| 发布负责人 | 决定是否回滚、暂停或继续发布 |
| QA 负责人 | 提供缺陷和风险证据 |
| 服务端负责人 | 执行镜像或配置回滚 |
| 客户端负责人 | 执行客户端版本回退或下架 |
| 运维负责人 | 确保环境恢复和监控恢复 |

## 4. 上线禁止项

1. P0 缺陷未清零。
2. 兼容矩阵没有覆盖关键运营商或平台。
3. 风险提示和帮助文案仍然误导。
4. 回滚路径未演练或不可用。
5. 关键依赖人未确认值守。

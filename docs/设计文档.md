## 1. 文档概述

### 1.1 目标

本文档是 `方案书.md` 的工程落地版本，提供可直接指导编码、联调、测试和部署的完整技术细节。

### 1.2 核心目标

| 目标 | 说明 |
|------|------|
| 隐私拨号 | 通过运营商支持的 CLIR 前缀（如 `#31#`）让被叫方看到"未知号码"或"私人号码" |
| 隐藏号码归属地 | 当 Caller ID 被隐藏时，被叫方无法通过来电信息推断主叫号码的地理归属 |
| 登录后使用 | 手机号验证码登录，未登录不可使用拨号功能 |
| 双平台 | Flutter 同时覆盖 Android 与 iOS |

### 1.3 关于"隐藏归属地"的技术说明

电话网络中，被叫方看到的来电信息来自 Caller ID（CLIP/CLIR）。当通过 CLIR 前缀（`#31#`）成功隐藏 Caller ID 时：

- 被叫方无法看到主叫号码
- 无号码则无法查询号码归属地（号段数据库依赖号码前 7 位）
- 被叫方手机会显示"未知号码""私人号码""来电号码已隐藏"等

**本方案通过隐藏完整 Caller ID 来间接实现隐藏归属地**，而非单独隐藏地区信息。电信网络不支持"只隐藏归属地但显示号码"这种部分隐藏。

### 3.2 路由设计

```dart
// lib/core/router/app_router.dart

final appRouter = GoRouter(
  initialLocation: '/splash',
  redirect: (context, state) {
    final authState = ref.read(authProvider);
    final isLoggedIn = authState.isAuthenticated;
    final isAuthRoute = state.matchedLocation.startsWith('/auth');
    final isSplash = state.matchedLocation == '/splash';

    if (isSplash) return null; // splash 自行判断跳转

    if (isLoggedIn && isAuthRoute) return '/home';

    GoRoute(path: '/auth/login',   builder: (_, __) => const LoginPage()),
    GoRoute(path: '/auth/verify',  builder: (_, __) => const VerifyCodePage()),
    GoRoute(path: '/home',         builder: (_, __) => const HomePage()),
    GoRoute(path: '/settings',     builder: (_, __) => const SettingsPage()),
    GoRoute(path: '/help',         builder: (_, __) => const HelpPage()),
```

### 3.3 状态管理（Riverpod）

#### 3.3.1 Auth Provider

```dart
// lib/features/auth/providers/auth_provider.dart

@riverpod
class Auth extends _$Auth {
  @override
  AuthState build() => const AuthState.unauthenticated();

  Future<void> sendCode(String phone) async {
    state = const AuthState.loading();
    try {
      await ref.read(authRepositoryProvider).sendCode(phone);
      state = AuthState.codeSent(phone: phone);
    } catch (e) {
      state = AuthState.error(e.toString());
    }
  }

  Future<void> verifyCode(String phone, String code) async {
    state = const AuthState.loading();
    try {
      final tokens = await ref.read(authRepositoryProvider).login(phone, code);
      await ref.read(secureStorageProvider).saveTokens(tokens);
      state = AuthState.authenticated(tokens: tokens);
    } catch (e) {
      state = AuthState.error(e.toString());
    }
  }

  Future<void> logout() async {
    try {
      await ref.read(authRepositoryProvider).logout();
    } finally {
      await ref.read(secureStorageProvider).clearTokens();
      state = const AuthState.unauthenticated();
    }
  }

  Future<bool> tryRefresh() async {
    try {
      final refreshToken = await ref.read(secureStorageProvider).getRefreshToken();
      if (refreshToken == null) return false;
      final tokens = await ref.read(authRepositoryProvider).refresh(refreshToken);
      await ref.read(secureStorageProvider).saveTokens(tokens);
      state = AuthState.authenticated(tokens: tokens);
      return true;
    } catch (_) {
      await ref.read(secureStorageProvider).clearTokens();
      state = const AuthState.unauthenticated();
      return false;
    }
  }
}
```

### 3.4 拨号核心逻辑

```dart
// lib/features/dialer/data/dial_service.dart

class DialService {
  /// 发起拨号
  /// [rawNumber] 用户输入的原始号码
  /// [prefix] 隐私前缀，如 "#31#"，普通拨打传空字符串
  Future<void> startCall(String rawNumber, String prefix) async {
    final normalized = _normalize(rawNumber);
    _validate(normalized);

    final dialString = '$prefix$normalized';
    final uri = Uri(scheme: 'tel', path: dialString);

    if (!await canLaunchUrl(uri)) {
      throw DialException('当前设备不支持电话拨号');
    }

    final launched = await launchUrl(uri, mode: LaunchMode.externalApplication);
    if (!launched) {
      throw DialException('无法调起系统拨号器');
    }
  }

  /// 号码清洗：去除空格、横杠、括号
  String _normalize(String input) {
    return input.replaceAll(RegExp(r'[\s\-\(\)（）]'), '');
  }

  /// 号码校验
  void _validate(String number) {
    // 中国大陆手机号：1开头 11位
    final cnMobile = RegExp(r'^1[3-9]\d{9}$');
    // 中国大陆座机：区号+号码，如 010-12345678
    final cnLandline = RegExp(r'^0\d{2,3}\d{7,8}$');
    // 国际号码：+开头
    final international = RegExp(r'^\+\d{7,15}$');

    if (!cnMobile.hasMatch(number) &&
        !cnLandline.hasMatch(number) &&
        !international.hasMatch(number)) {
      throw DialException('号码格式不正确');
    }
  }
}
```

#### 3.5.2 Auth 拦截器（自动刷新 Token）

```dart
// lib/core/network/auth_interceptor.dart

class AuthInterceptor extends Interceptor {
  final Ref ref;
  AuthInterceptor(this.ref);

  @override
  void onRequest(RequestOptions options, RequestInterceptorHandler handler) async {
    // 登录相关接口不需要 Token
    if (_isAuthEndpoint(options.path)) {
      handler.next(options);
      return;
    }

    final token = await ref.read(secureStorageProvider).getAccessToken();
    if (token != null) {
      options.headers['Authorization'] = 'Bearer $token';
    }
    handler.next(options);
  }

  @override
  void onError(DioException err, ErrorInterceptorHandler handler) async {
    if (err.response?.statusCode == 401) {
      // 尝试刷新 Token
      final refreshed = await ref.read(authProvider.notifier).tryRefresh();
      if (refreshed) {
        // 用新 Token 重试原请求
        final token = await ref.read(secureStorageProvider).getAccessToken();
        err.requestOptions.headers['Authorization'] = 'Bearer $token';
        final response = await Dio().fetch(err.requestOptions);
        handler.resolve(response);
        return;
      }
    }
    handler.next(err);
  }

  bool _isAuthEndpoint(String path) {
    return path.contains('/auth/send-code') ||
           path.contains('/auth/login') ||
           path.contains('/auth/refresh');
  }
}
```

### 3.6 本地存储方案

| 数据 | 存储方式 | 说明 |
|------|----------|------|
| Access Token | `flutter_secure_storage` | 加密存储，Android 使用 EncryptedSharedPreferences，iOS 使用 Keychain |
| Refresh Token | `flutter_secure_storage` | 同上 |
| 用户手机号 | `flutter_secure_storage` | 脱敏显示，完整号码加密存储 |
| 默认隐私前缀 | `shared_preferences` | 非敏感配置 |
| 是否已阅读提示 | `shared_preferences` | 非敏感配置 |
| 最近拨打号码 | `shared_preferences` | JSON 数组，最多保存 50 条，仅存号码后 4 位脱敏显示用 + 完整号码 |

```dart
// lib/core/storage/secure_storage.dart

class SecureStorageService {
  final FlutterSecureStorage _storage = const FlutterSecureStorage(
    aOptions: AndroidOptions(encryptedSharedPreferences: true),
  );

  static const _accessTokenKey = 'access_token';
  static const _refreshTokenKey = 'refresh_token';

  Future<void> saveTokens(TokenPair tokens) async {
    await Future.wait([
      _storage.write(key: _accessTokenKey, value: tokens.accessToken),
      _storage.write(key: _refreshTokenKey, value: tokens.refreshToken),
    ]);
  }

  Future<String?> getAccessToken() => _storage.read(key: _accessTokenKey);
  Future<String?> getRefreshToken() => _storage.read(key: _refreshTokenKey);

  Future<void> clearTokens() async {
    await Future.wait([
      _storage.delete(key: _accessTokenKey),
      _storage.delete(key: _refreshTokenKey),
    ]);
  }
}
```

### 3.7 平台差异处理

#### 3.7.1 Android 配置

```xml
<!-- android/app/src/main/AndroidManifest.xml -->

<manifest>
    <uses-permission android:name="android.permission.INTERNET" />

    <application ...>
        <!-- Android 11+ 需要声明 tel scheme 查询能力 -->
        <queries>
            <intent>
                <action android:name="android.intent.action.DIAL" />
                <data android:scheme="tel" />
            </intent>
        </queries>
    </application>
</manifest>
```

**注意**：MVP 阶段不申请 `CALL_PHONE` 权限。使用 `ACTION_DIAL`（调起拨号器由用户确认）而非 `ACTION_CALL`（直接拨打）。

#### 3.7.2 iOS 配置

```xml
<!-- ios/Runner/Info.plist -->

<key>LSApplicationQueriesSchemes</key>
<array>
    <string>tel</string>
</array>
```

#### 3.7.3 平台差异对照表

| 行为 | Android | iOS |
|------|---------|-----|
| `tel:#31#number` 调起方式 | 打开系统拨号器，前缀正常显示 | 打开 Phone App，部分机型可能裁剪 `#` |
| 用户确认 | 系统拨号器界面需用户点击拨打 | 弹出确认对话框后拨打 |
| `#31#` 处理 | 多数机型与运营商可正常处理 | 依赖运营商，部分可能被系统忽略 |
| 权限要求 | 仅 INTERNET（不需要 CALL_PHONE） | 无额外权限 |
| 备选方案 | `*31#number`（部分运营商） | 引导用户在设置中手动关闭 Caller ID |

#### 3.8.1 启动页 (SplashPage)

```
流程：
1. 显示 App Logo + 加载动画
2. 并行执行：
   a. 从 SecureStorage 读取 Access Token
   b. 从服务端拉取最新配置（前缀列表、公告）
3. Token 有效 → 导航到 /home
4. Token 过期 → 尝试 Refresh
   a. Refresh 成功 → /home
   b. Refresh 失败 → /auth/login
5. 无 Token → /auth/login
```

### 4.2 模块设计

#### 4.2.1 Auth Module

**职责**：验证码发送、登录验证、Token 签发与刷新、登出

```typescript
// src/modules/auth/auth.service.ts

@Injectable()
export class AuthService {
  constructor(
    private readonly smsService: SmsService,
    private readonly userService: UserService,
    private readonly jwtService: JwtService,
    private readonly rateLimitService: RateLimitService,
    @InjectRepository(AuthCode) private authCodeRepo: Repository<AuthCode>,
    @InjectRepository(RefreshToken) private refreshTokenRepo: Repository<RefreshToken>,
  ) {}

  /**
   * 发送验证码
   * 1. 限流检查（手机号维度、IP 维度）
   * 2. 生成 6 位随机码
   * 3. 存储验证码哈希 + 过期时间
   * 4. 调用短信服务下发
   */
  async sendCode(phone: string, ip: string): Promise<void> {
    await this.rateLimitService.checkSendCode(phone, ip);

    const code = this.generateCode();
    const codeHash = await bcrypt.hash(code, 10);
    const expiredAt = new Date(Date.now() + 5 * 60 * 1000); // 5 分钟有效

    await this.authCodeRepo.save({
      phone,
      codeHash,
      expiredAt,
      sendIp: ip,
    });

    await this.smsService.send(phone, code);
  }

  /**
   * 验证码登录
   * 1. 查找最新未使用、未过期的验证码
   * 2. 比对哈希
   * 3. 标记验证码已使用
   * 4. 创建或查找用户
   * 5. 签发 Access Token + Refresh Token
   */
  async login(phone: string, code: string, deviceId: string): Promise<TokenPair> {
    const authCode = await this.authCodeRepo.findOne({
      where: { phone, usedAt: IsNull() },
      order: { createdAt: 'DESC' },
    });

    if (!authCode || authCode.expiredAt < new Date()) {
      throw new UnauthorizedException('验证码已过期');
    }

    const isValid = await bcrypt.compare(code, authCode.codeHash);
    if (!isValid) {
      await this.rateLimitService.recordFailedAttempt(phone);
      throw new UnauthorizedException('验证码错误');
    }

    // 标记验证码已使用
    authCode.usedAt = new Date();
    await this.authCodeRepo.save(authCode);

    // 创建或查找用户
    const user = await this.userService.findOrCreate(phone);

    // 签发 Token
    return this.issueTokens(user.id, deviceId);
  }

  /**
   * 刷新 Token
   * 1. 验证 Refresh Token 哈希
   * 2. 检查是否过期或已注销
   * 3. 注销旧 Refresh Token（轮换）
   * 4. 签发新的 Token 对
   */
  async refresh(refreshToken: string, deviceId: string): Promise<TokenPair> {
    const tokenHash = this.hashToken(refreshToken);
    const stored = await this.refreshTokenRepo.findOne({
      where: { tokenHash, revokedAt: IsNull() },
    });

    if (!stored || stored.expiredAt < new Date()) {
      throw new UnauthorizedException('Refresh Token 无效');
    }

    // 轮换：注销旧 Token
    stored.revokedAt = new Date();
    await this.refreshTokenRepo.save(stored);

    return this.issueTokens(stored.userId, deviceId);
  }

  private async issueTokens(userId: string, deviceId: string): Promise<TokenPair> {
    const accessToken = this.jwtService.sign(
      { sub: userId },
      { expiresIn: '2h' },
    );

    const refreshToken = crypto.randomBytes(32).toString('hex');
    const tokenHash = this.hashToken(refreshToken);

    await this.refreshTokenRepo.save({
      userId,
      tokenHash,
      deviceId,
      expiredAt: new Date(Date.now() + 30 * 24 * 60 * 60 * 1000), // 30 天
    });

    return { accessToken, refreshToken };
  }

  private generateCode(): string {
    return crypto.randomInt(100000, 999999).toString();
  }

  private hashToken(token: string): string {
    return crypto.createHash('sha256').update(token).digest('hex');
  }
}
```

#### 4.2.3 Rate Limit Module

```typescript
// src/modules/rate-limit/rate-limit.service.ts

@Injectable()
export class RateLimitService {
  constructor(@InjectRedis() private redis: Redis) {}

  /**
   * 验证码发送限流
   * - 同一手机号：60 秒 1 次，1 小时 5 次
   * - 同一 IP：1 小时 20 次
   */
  async checkSendCode(phone: string, ip: string): Promise<void> {
    // 60 秒内不可重复发送
    const recentKey = `sms:recent:${phone}`;
    const exists = await this.redis.exists(recentKey);
    if (exists) {
      throw new TooManyRequestsException('请 60 秒后再试');
    }

    // 1 小时内最多 5 次
    const hourlyKey = `sms:hourly:${phone}`;
    const hourlyCount = await this.redis.incr(hourlyKey);
    if (hourlyCount === 1) {
      await this.redis.expire(hourlyKey, 3600);
    }
    if (hourlyCount > 5) {
      throw new TooManyRequestsException('验证码发送次数已达上限，请 1 小时后再试');
    }

    // IP 维度限流
    const ipKey = `sms:ip:${ip}`;
    const ipCount = await this.redis.incr(ipKey);
    if (ipCount === 1) {
      await this.redis.expire(ipKey, 3600);
    }
    if (ipCount > 20) {
      throw new TooManyRequestsException('请求过于频繁');
    }

    // 设置 60 秒冷却
    await this.redis.set(recentKey, '1', 'EX', 60);
  }

  /**
   * 记录验证码输入错误次数
   * 连续错误 5 次后锁定 30 分钟
   */
  async recordFailedAttempt(phone: string): Promise<void> {
    const key = `sms:fail:${phone}`;
    const count = await this.redis.incr(key);
    if (count === 1) {
      await this.redis.expire(key, 1800); // 30 分钟
    }
    if (count >= 5) {
      throw new TooManyRequestsException('验证码错误次数过多，请 30 分钟后再试');
    }
  }
}
```

## 7. 号码归属地隐藏策略

### 7.1 原理说明

移动电话网络中，被叫方设备通过 **Caller ID (CLIP)** 功能获取主叫号码。号码归属地信息并非单独传输的字段，而是被叫方设备或运营商根据号码前缀（号段）在本地数据库中查询得到的。

**隐藏归属地的唯一可靠方式是隐藏完整号码**。当 CLIR（Calling Line Identification Restriction）生效时：

1. 主叫号码不会传递给被叫方
2. 被叫方设备无号码可查，无法显示归属地
3. 被叫方看到"未知号码""私人号码"或"来电号码已隐藏"

### 7.2 各国/运营商 CLIR 前缀对照表

| 国家/地区 | 前缀 | 标准 | 适用运营商 | 备注 |
|-----------|------|------|-----------|------|
| 中国大陆 | `#31#` | GSM/3GPP | 移动、联通（GSM/LTE） | 电信 CDMA 网络支持度低 |
| 美国/加拿大 | `*67` | NANP | AT&T, Verizon, T-Mobile 等 | 固话和手机通用 |
| 英国 | `141` | BT 标准 | 多数运营商 | 通用性好 |
| 中国香港 | `133` | 本地标准 | 多数运营商 | — |
| 日本 | `184` | NTT 标准 | NTT, KDDI, SoftBank | 通用性好 |
| 韩国 | `*23#` | 本地标准 | SKT, KT, LG U+ | — |
| 德国 | `#31#` | GSM/3GPP | 通用 | — |
| 法国 | `#31#` | GSM/3GPP | 通用 | — |
| 澳大利亚 | `#31#` | GSM/3GPP | 通用 | — |
| 印度 | `#31#` | GSM/3GPP | 部分运营商 | 效果不稳定 |

### 7.3 中国三大运营商详细支持情况

| 运营商 | 网络制式 | `#31#` 支持 | 备注 |
|--------|---------|-------------|------|
| 中国移动 | GSM / LTE / 5G NR | **支持**（多数地区有效） | VoLTE 下效果较好 |
| 中国联通 | GSM / WCDMA / LTE / 5G NR | **支持**（多数地区有效） | 效果与移动相当 |
| 中国电信 | CDMA / LTE / 5G NR | **部分支持** | CDMA 网络可能不识别 `#31#`；VoLTE 下效果改善 |

**重要提醒**：

- `#31#` 是 GSM/3GPP 标准的 CLIR 补充业务前缀，在 GSM/LTE/NR 网络上有较好的标准支持
- 中国电信的传统 CDMA 网络使用 IS-41 标准，不原生支持 `#31#`
- 随着 VoLTE/VoNR 的普及，三大运营商的 CLIR 支持度在持续改善
- 部分运营商在特定地区可能通过 BOSS 系统限制了 CLIR 功能
- 效果不稳定时，可建议用户联系运营商开通"主叫号码隐藏"业务

### 7.4 隐藏归属地效果对照

| 场景 | 被叫方看到的信息 | 归属地是否可见 |
|------|-----------------|---------------|
| 正常拨打 | `13812345678` | 可见（北京移动） |
| `#31#` 生效 | "未知号码" / "私人号码" | **不可见** |
| `#31#` 被运营商忽略 | `13812345678` | 可见 |
| 运营商侧开通号码隐藏服务 | "未知号码" | **不可见** |

### 7.5 客户端前缀选择逻辑

```dart
// lib/features/dialer/data/dial_service.dart

class PrefixSelector {
  /// 根据用户配置和服务端前缀列表选择最佳前缀
  ///
  /// 优先级：
  /// 1. 用户手动设置的前缀
  /// 2. 服务端按运营商匹配的前缀
  /// 3. 服务端按国家通用的前缀
  /// 4. 兜底默认 #31#
  String selectPrefix({
    required String? userPreference,
    required List<DialPrefix> serverPrefixes,
    required String countryCode,
    String? carrierName,
  }) {
    // 用户手动设置优先
    if (userPreference != null && userPreference.isNotEmpty) {
      return userPreference;
    }

    // 按运营商匹配
    if (carrierName != null) {
      final carrierMatch = serverPrefixes
          .where((p) => p.countryCode == countryCode && p.carrierName == carrierName)
          .toList()
        ..sort((a, b) => b.priority.compareTo(a.priority));
      if (carrierMatch.isNotEmpty) return carrierMatch.first.prefix;
    }

    // 按国家通用匹配
    final countryMatch = serverPrefixes
        .where((p) => p.countryCode == countryCode && p.carrierName == '*')
        .toList()
      ..sort((a, b) => b.priority.compareTo(a.priority));
    if (countryMatch.isNotEmpty) return countryMatch.first.prefix;

    // 兜底
    return '#31#';
  }
}
```

### 7.6 失败检测与回退机制

App 本身无法直接检测 CLIR 是否生效（结果取决于被叫方运营商的反馈），但可以通过以下方式辅助：

```
用户点击"隐私拨打"
    │
    ├── 首次使用 → 弹出提示弹窗
    │   "隐私拨打效果取决于运营商，无法保证 100% 有效"
    │   [✓ 不再提示]  [取消] [确认拨打]
    │
    ├── 调起拨号器成功
    │   └── 通话结束后 → 展示反馈入口
    │       "对方是否显示了你的号码？"
    │       [是，仍然显示] → 引导：
    │           1. 尝试切换前缀
    │           2. 联系运营商开通号码隐藏
    │           3. 使用普通拨打
    │       [否，已隐藏] → 记录为有效前缀
    │
    └── 调起拨号器失败
        └── 提示：当前设备不支持拨号功能
            [使用普通拨打] [取消]
```

## 8. 安全设计

### 8.1 Token 安全

| 措施 | 说明 |
|------|------|
| Access Token 存储 | `flutter_secure_storage`，Android EncryptedSharedPreferences / iOS Keychain |
| Refresh Token 存储 | 同上 |
| Refresh Token 轮换 | 每次刷新后旧 Token 立即注销，签发新 Token |
| Token 传输 | 仅通过 HTTPS 传输 |
| JWT 签名算法 | HS256（对称）或 RS256（非对称），生产环境推荐 RS256 |

### 8.2 验证码安全

| 措施 | 说明 |
|------|------|
| 验证码存储 | bcrypt 哈希，不存明文 |
| 有效期 | 5 分钟 |
| 使用次数 | 一次性，使用后标记 `used_at` |
| 发送限流 | 同号码 60s/次，1h/5次；同 IP 1h/20次 |
| 输入限流 | 连续错误 5 次锁定 30 分钟 |

### 8.3 数据脱敏

| 场景 | 脱敏规则 |
|------|----------|
| API 返回手机号 | `138****8000`（中间 4 位） |
| 服务端日志 | 手机号脱敏为 `138****8000` |
| 错误上报（Sentry） | 手机号、验证码、Token 均不上报 |
| 客户端本地历史 | 列表显示脱敏号码，拨号时使用完整号码 |

### 8.4 传输安全

| 措施 | 说明 |
|------|------|
| HTTPS | 服务端强制 HTTPS，HTTP 301 重定向 |
| 证书锁定 | 生产环境可选开启 SSL Pinning（Dio 支持） |
| 请求签名 | MVP 阶段可选，后续可增加 HMAC 请求签名防篡改 |

### 8.5 服务端安全

| 措施 | 说明 |
|------|------|
| SQL 注入防护 | TypeORM 参数化查询 |
| 全局限流 | `@nestjs/throttler` 全局限流 |
| CORS | 仅允许 App 请求（移动端无 CORS 问题，管理后台需配置） |
| Helmet | `helmet` 中间件设置安全响应头 |
| 密钥管理 | 环境变量 + Secret Manager，不硬编码 |
